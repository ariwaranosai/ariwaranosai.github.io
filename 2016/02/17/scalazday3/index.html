Kinds and some type-foo

之前的内容只关注构建自己的Types以及Typeclasses但是并没有特别讨论kinds以及types。

Types可以看做values上带着的小标签，我们通过这个标签来获取一些诸如判断如何操作使用的信息。而Types本身也有这样的标签叫做Kinds。

在ghci里面使用:k命令可以可以查看type的kind。在scala console里，也可以同样查看类型。

从最上面开始,Int以及其他具体类型，都用*表示，在scala里面采用A来表示，first-order value，像(_: Int) + 3这种函数，或者如Option、Either这种type constructor，我们使用诸如*->*或者*->*->*。值得注意的是，Option[Int]就是*，而Option才是*->*。用scala的类型变量声明就是F[+A]，F[+A1,+A2]。

higher-order value像(f: Int => Int, list: List[Int]) => list map {f}这种高阶函数，是一种接受其他type constructor的type constructor。类型如(* -> *) -> *，或者可以写成X[F[A]]。

在scalaz 7.1中，Equal以及其他一些的类型的kind为F[A]，而Functor以及其他从它派生而来的类型的kind为X[F[A]]。Scala倾向于交叉着使用type class和type constructor，一些术语显得有些混乱。比如，List本身符合Functor的性质，而事实上Functor[List]可以从List派生出来，所以我们说List is a functor。

In FP, "is a" means "a instance can be derived from." @jimduey # CPL14 It's a provable relationship, not reliant on LSP.
